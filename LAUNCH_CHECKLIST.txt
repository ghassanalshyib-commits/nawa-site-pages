قائمة التحقق قبل رفع موقع نواة على GitHub Pages:

1) ارفع ملفات الموقع إلى مستودع GitHub بدون ملف ZIP.
2) من GitHub اختر:
   Settings > Pages > Deploy from a branch
   Branch: main
   Folder: /root
3) رابط الموقع المتوقع:
   https://ghassanalshyib-commits.github.io/nawa-site-pages/
4) في Supabase > Authentication > URL Configuration:
   Site URL:
   https://ghassanalshyib-commits.github.io/nawa-site-pages/

   Redirect URLs:
   https://ghassanalshyib-commits.github.io/nawa-site-pages/**
   https://ghassanalshyib-commits.github.io/nawa-site-pages/reset-password.html
5) في Supabase > Email Templates > Reset Password استخدم قالب:
   SUPABASE_RESET_PASSWORD_EMAIL_TEMPLATE.html
6) نفذ ملفات SQL داخل Supabase SQL Editor:
   - supabase-portfolios.sql
   - supabase-security-hardening.sql
7) اختبر:
   - إنشاء حساب ببريد حقيقي
   - تسجيل الدخول
   - حفظ محفظة
   - حذف محفظة
   - استرجاع كلمة المرور
8) تم ضبط robots.txt وsitemap.xml على رابط GitHub Pages الحالي.
9) لا تضع service_role key داخل ملفات الموقع. استخدم anon public key فقط.
